# TinyDB 存储引擎设计文档（阶段二）

## 1. 概述

本文档描述了 TinyDB 阶段二的存储引擎设计，包括页式存储、缓冲区管理、WAL日志和系统表的实现。

## 2. 存储引擎架构

```
┌─────────────────────────────────────────┐
│           SQL 执行层 (Executor)          │
├─────────────────────────────────────────┤
│           表管理器 (TableManager)        │
│  ┌───────────────────────────────────┐  │
│  │  系统表: tn_class, tn_attribute   │  │
│  └───────────────────────────────────┘  │
├─────────────────────────────────────────┤
│           存储引擎 (StorageEngine)       │
│  ┌─────────────┐  ┌─────────────────┐  │
│  │  缓冲池管理  │  │    WAL管理器     │  │
│  │ BufferPool  │  │   WALManager    │  │
│  └─────────────┘  └─────────────────┘  │
├─────────────────────────────────────────┤
│           磁盘管理器 (DiskManager)       │
├─────────────────────────────────────────┤
│           数据库文件 (tinydb.db)         │
└─────────────────────────────────────────┘
```

## 3. 页式存储设计

### 3.1 页格式

TinyDB 使用 8KB 定长页，页结构如下：

```
+----------------------------------+
|           Page Header            |  32 bytes
|  - page_id (4 bytes)             |
|  - next_page_id (4 bytes)        |
|  - free_space (2 bytes)          |
|  - free_space_size (2 bytes)     |
|  - tuple_count (2 bytes)         |
|  - flags (2 bytes)               |
|  - checksum (4 bytes)            |
|  - lsn (4 bytes)                 |
+----------------------------------+
|           Tuple Data             |  从页头后开始，向下增长
|  ┌─────────┐                     |
|  │ Tuple 1 │                     |
|  ├─────────┤                     |
|  │ Tuple 2 │                     |
|  │   ...   │                     |
|  └─────────┘                     |
+----------------------------------+
|           Free Space             |
+----------------------------------+
|         Slot Directory           |  从页尾开始，向上增长
|  ┌─────────┐                     |
|  │ Slot N  │                     |
|  ├─────────┤                     |
|  │ Slot 2  │                     |
|  │ Slot 1  │                     |
|  │ Slot 0  │                     |
|  └─────────┘                     |
+----------------------------------+
```

### 3.2 Slot Entry 结构

```cpp
struct SlotEntry {
    uint16_t offset;    // 元组偏移量
    uint16_t length;    // 元组长度
    uint16_t flags;     // 标志位 (IN_USE, DELETED)
};
```

### 3.3 文件格式

数据库文件结构：

```
+----------------------------------+
|          File Header             |  第0页
|  - magic: "TINYDB\0\0"           |
|  - version: 1                    |
|  - page_size: 8192               |
|  - page_count                    |
|  - free_page (空闲页链表头)       |
+----------------------------------+
|           Page 1                 |
|         (tn_class 数据)          |
+----------------------------------+
|           Page 2                 |
|       (tn_attribute 数据)        |
+----------------------------------+
|           Page 3+                |
|         (用户表数据)              |
+----------------------------------+
```

## 4. 缓冲区管理器

### 4.1 缓冲帧 (BufferFrame)

```cpp
class BufferFrame {
    PageId page_id_;        // 页号
    Page page_;             // 页数据
    int pin_count_;         // 引用计数
    bool is_dirty_;         // 脏页标志
};
```

### 4.2 LRU 替换策略

- 使用双向链表维护页的访问顺序
- 最近访问的页移到链表头部
- 需要替换时，从链表尾部选择未固定的页
- 使用哈希表实现 O(1) 的查找和删除

### 4.3 主要操作

| 操作 | 说明 |
|-----|------|
| fetchPage | 获取页，如果不存在则从磁盘加载 |
| newPage | 分配新页 |
| unpinPage | 释放页，减少引用计数 |
| flushPage | 将脏页写回磁盘 |
| flushAllPages | 刷新所有脏页 |

## 5. WAL 日志设计

### 5.1 日志记录格式

```
+----------------------------------+
|         Log Record Header        |  28 bytes
|  - size (4 bytes)                |
|  - type (1 byte)                 |
|  - txn_id (4 bytes)              |
|  - lsn (8 bytes)                 |
|  - prev_lsn (8 bytes)            |
|  - checksum (4 bytes)            |
+----------------------------------+
|          Log Record Data         |  可变长度
+----------------------------------+
```

### 5.2 日志类型

| 类型 | 说明 |
|-----|------|
| BEGIN | 事务开始 |
| COMMIT | 事务提交 |
| ROLLBACK | 事务回滚 |
| INSERT | 插入记录 |
| DELETE | 删除记录 |
| UPDATE | 更新记录 |
| NEW_PAGE | 新页分配 |
| CHECKPOINT | 检查点 |

### 5.3 WAL 协议

1. **写日志优先**：任何数据修改前先写日志
2. **日志刷盘**：事务提交前必须将日志刷到磁盘
3. **顺序写**：日志追加写入，保证顺序性

## 8. 行格式设计

### 8.1 元组存储格式

```
+----------------------------------+
|         NULL Bitmap              |  (n+7)/8 bytes
|  每列1位，表示是否为NULL          |
+----------------------------------+
|         Column Data 1            |
+----------------------------------+
|         Column Data 2            |
+----------------------------------+
|              ...                 |
+----------------------------------+
```

### 8.2 字段存储格式

定长类型（INT32, INT64, FLOAT, DOUBLE, BOOL）：
- 直接存储原始字节

变长类型（CHAR, VARCHAR）：
```
+----------------------------------+
|       Length (4 bytes)           |
+----------------------------------+
|       Data (length bytes)        |
+----------------------------------+
```

## 9. TID（元组标识符）

```cpp
struct TID {

```

TID 用于唯一标识表中的一行数据，支持快速定位。

## 10. 关键流程

### 10.1 创建表流程

1. 分配新的数据页
2. 创建 TableMeta 对象
3. 将表元数据写入 tn_class
4. 将列元数据写入 tn_attribute
5. 在内存中缓存表信息

### 10.2 插入数据流程

1. 获取表的元数据
2. 序列化元组数据
3. 查找或分配有足够空间的页
4. 在页中插入元组数据
5. 更新页头和槽位目录
6. 标记页为脏页
7. 写 WAL 日志（阶段二简化实现）

### 10.3 查询数据流程

1. 获取表的元数据
2. 从第一个数据页开始遍历
3. 对每个页，遍历所有有效槽位
4. 反序列化元组数据
5. 返回结果

## 12. 性能考虑

1. **缓冲池**：使用 LRU 策略缓存热点数据页
2. **顺序写**：WAL 日志追加写入，提高写入性能
3. **延迟刷盘**：脏页延迟刷盘，减少 I/O 次数
4. **页内存储**：元组紧凑存储，减少空间浪费

## 13. 限制与待改进

1. **阶段二限制**：
   - 暂不支持索引
   - 暂不支持事务隔离级别
   - 暂不支持并发控制
   - 恢复机制仅提供接口

2. **未来改进**：
   - 实现 B+ 树索引
   - 完整的事务管理（ACID）
   - 并发控制和锁管理
   - 崩溃恢复机制
